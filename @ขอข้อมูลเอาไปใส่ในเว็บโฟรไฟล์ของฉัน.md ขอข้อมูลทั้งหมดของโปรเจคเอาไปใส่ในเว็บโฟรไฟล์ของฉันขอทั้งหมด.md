## 1. ชื่อและภาพรวม

| รายการ | รายละเอียด |

## 4. ฟีเจอร์หลัก (แบบละเอียด)

### 4.1 สินค้าและร้านค้า
- **สินค้า:** CRUD, หมวดหมู่, รูปหลายรูป (Cloudinary), ตัวเลือก variant (สี/ไซส์, ProductVariant), ราคาปกติ/ลดราคา (discountPrice, discountStartDate, discountEndDate), สต็อก
- **ร้านค้า:** สมัครร้าน (Seller Onboarding), แอดมินอนุมัติ/ระงับ (status: pending/approved/suspended), กระเป๋าร้าน (StoreWallet, StoreWalletTransaction), รายงานยอดขาย
- **การค้นหา:** search/filters, รายการสินค้า, สินค้าลดราคา, สินค้า IT

### 4.2 ตะกร้าและคำสั่งซื้อ
- **ตะกร้า:** เพิ่ม/ลดจำนวน, เลือก variant (selectedVariants), คำนวณรวม, ลบรายการ, ล้างตะกร้า
- **สร้างออเดอร์:** ที่อยู่จัดส่ง, โทรศัพท์, โค้ดส่วนลด (คูปอง), ค่าขนส่ง (preview-shipping), สร้าง Order + ProductOnOrder
- **Flow พิเศษ:** กด "ซื้อสินค้า" จาก ProductDetail → ส่ง state buyNow ไปหน้า cart → ติ๊กถูกสินค้านั้นอัตโนมัติ

### 4.3 การชำระเงิน
- **วิธีชำระ:** เงินสด (COD), บัตรเครดิต/เดบิต (mock), QR Code (PromptPay)
- **QR:** createPayment (method: qr_code) → ได้ transactionId, qrCodeData; generatePromptPayQR; หน้าชำระเงินโพลสถานะทุก 2 วินาที
- **อัปโหลดสลิป:** uploadPaymentSlip → สถานะรอแอดมิน; แอดมิน approve/reject
- **SMS Webhook:** `POST /api/payments/webhook/sms` — Body: transactionId/orderId หรือ sender, message, receivedAt; จับคู่ payment รอชำระ (หรือ Fallback: รายการล่าสุด); อัปเดต payment = completed, order = PENDING_CONFIRMATION; ส่ง Socket.IO ให้ลูกค้า
- **Wallet ลูกค้า:** Wallet, WalletTransaction (DEPOSIT/WITHDRAW/PAYMENT/REFUND); คืนเงินเมื่อยกเลิก/คืนสินค้า; จ่ายด้วย Wallet ได้

### 4.4 Logistics (การจัดส่ง)
- **OrderStatus (enum):** PENDING → VERIFYING → PENDING_CONFIRMATION → PROCESSING → READY_FOR_PICKUP → RIDER_ASSIGNED → PICKED_UP → SHIPPED → OUT_FOR_DELIVERY → DELIVERED → COMPLETED (และ CANCELLED, REFUND_REQUESTED, REFUNDED)
- **ShipmentStatus (enum):** WAITING_PICKUP → IN_TRANSIT → OUT_FOR_DELIVERY → DELIVERED / FAILED
- **Shipment:** สร้างอัตโนมัติเมื่อร้านอัปเดต order เป็น READY_FOR_PICKUP; 1:1 กับ Order
- **ไรเดอร์:** available-jobs → assign (assignCourier) → pickup → out-for-delivery → complete (อัปโหลด proofImage ได้); อัปเดต Order + TrackingHistory
- **ลูกค้า:** ดู TrackingTimeline, confirm-received → COMPLETED

### 4.5 การคืนสินค้า / คืนเงิน (Return & Refund)
- **เงื่อนไข:** ออเดอร์ DELIVERED/COMPLETED และภายใน 7 วัน (receivedAt หรือ shipment.deliveredTime)
- **refundStatus (Order):** NONE | REQUESTED | WAITING_FOR_PICKUP | RETURN_IN_TRANSIT | RETURN_DELIVERED | IN_DISPUTE | APPROVED | REJECTED
- **Flow:** ลูกค้า POST return (เหตุผล, returnItems, proof) → ร้าน return-call-rider → สร้าง/อัปเดต ReturnPickup → ไรเดอร์ return-jobs assign → pickup → complete (proof) → ร้าน return-approve (คืนเงินเข้า Wallet, หัก Store, คืนสต็อก)
- **Dispute:** ร้าน return-dispute (เหตุผล + รูป) → IN_DISPUTE → แอดมิน Dispute Management → return-approve (อนุมัติ/ปฏิเสธ)

### 4.6 แอดมิน
- แดชบอร์ด (Analytics), ผู้ใช้งาน (UserManagement), อนุมัติร้านค้า (StoreApproval), สินค้า (ProductManagement), หมวดหมู่ (CategoryManagement), แบนเนอร์ (BannerManagement), คำสั่งซื้อ (OrderManagement), คำขอคืน (ReturnManagement), จัดการข้อพิพาท (DisputeManagement), การชำระเงิน (PaymentApproval), ตั้งค่าเว็บ (SiteSettings)

### 4.7 ไรเดอร์
- งานที่รอรับ (available-jobs: จัดส่ง + return-jobs); งานของฉัน (my); รับงาน → รับของที่ร้าน → กำลังนำจ่าย → ส่งสำเร็จ; งานรับของคืน: รับงาน → รับของจากลูกค้า → ส่งของคืนร้าน (อัปโหลด proof)

### 4.8 อื่นๆ
- การแจ้งเตือน (Notification), Socket.IO (order_status_updated, payment); คูปอง (validate, my-coupons); รีวิว (create, get by product/store, delete); AI ถามคำถามสินค้า (product-question)

## 10. การรันโปรเจค (ขั้นตอนละเอียด)

1. ติดตั้ง MySQL และสร้าง database
2. ในโฟลเดอร์ `server`: สร้างไฟล์ `.env` มี `DATABASE_URL="mysql://USER:PASSWORD@HOST:PORT/DATABASE"`
3. `cd server && npm install && npx prisma generate` (และ migrate ถ้าต้องการ)
4. `npm start` ใน server → รันที่พอร์ต 3000
5. `cd client && npm install` แล้วตั้ง `proxy` ใน package.json เป็น `http://localhost:3000` (มีอยู่แล้ว)
6. `npm start` ใน client → เปิด http://localhost:3001
7. (ถ้าต้องการ Webhook SMS) รัน `ngrok http 3000` แล้วใช้ URL ไปตั้งในแอปดัก SMS → POST `https://<ngrok>/api/payments/webhook/sms`

## 13. สรุปสำหรับใส่ในเว็บโปรไฟล์

### ฉบับสั้น
**BoxiFY** — ร้านค้าออนไลน์ Full-stack (React, Node.js, Express, Prisma, MySQL). หลายบทบาท: ลูกค้า/ร้าน/ไรเดอร์/แอดมิน. ฟีเจอร์: สั่งซื้อ, ชำระเงิน QR/สลิป/Webhook SMS, Logistics ไรเดอร์, คืนสินค้าและ Dispute, แดชบอร์ดแอดมิน, Wallet, คูปอง, รีวิว. ใช้ Socket.IO Realtime, JWT, PromptPay QR.

### ฉบับยาว
**BoxiFY** เป็นโปรเจค E-commerce แบบครบวงจร พัฒนาด้วย **React 18** (Frontend) และ **Node.js + Express 5 + Prisma** (Backend) ใช้ **MySQL** เก็บข้อมูล และ **Socket.IO** สำหรับ Realtime. ระบบรองรับ 4 บทบาท: **ลูกค้า** (สั่งซื้อ, ชำระเงิน QR PromptPay/สลิป/Webhook SMS, ตะกร้า, ติดตามออเดอร์, ขอคืนสินค้า, Wallet), **ร้านค้า** (จัดการร้าน/สินค้า/ออเดอร์, กระเป๋าร้าน, จัดการคำขอคืนและข้อพิพาท), **ไรเดอร์** (รับงานจัดส่งและรับของคืน, อัปเดตสถานะจนส่งสำเร็จ), **แอดมิน** (อนุมัติร้าน, จัดการผู้ใช้/สินค้า/หมวดหมู่/แบนเนอร์/คำสั่งซื้อ/คำขอคืน/ข้อพิพาท/การชำระเงิน, Analytics, ตั้งค่าเว็บ). มีฟีเจอร์ Logistics เต็มรูปแบบ (OrderStatus, Shipment, ReturnPickup), การคืนสินค้าพร้อม Dispute, แดชบอร์ดและ API ครบ. ใช้ JWT, Passport (Google/Facebook), Multer/Cloudinary, promptpay-qr, Chart.js, React Toastify, Motion. พัฒนาแบบ Full-stack ครอบคลุมการออกแบบ DB, REST API, Realtime, และ Webhook ชำระเงิน.
